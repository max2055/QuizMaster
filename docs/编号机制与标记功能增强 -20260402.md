## 📋 需求背景

### 1️⃣ 标记功能用途
用户有两个使用场景：
- **标记疑难题目**：在答题卡上显示特殊标识，方便后续重点复习
- **标记已掌握题目**：与错题本的"已掌握"功能类似，用于标记已掌握的错题

### 2️⃣ 编号机制问题
题目从原始 DOC 转成 CSV 后导入，用户希望：
- 单选题从 1 开始连续编号（1-528）
- 多选题从 1 开始连续编号（1-400）
- 显示时使用连续编号，而非数据库递增 ID

### 2️⃣ 标记功能增强

**标记状态显示**：
- 答题卡按钮：标记题目显示 📌 标识（如 `15📌`）
- 标记按钮：显示"📌 标记"或"📌 已标记"

**标记持久化**：
- 标记状态保存到 `practice_records` 表
- 重启应用后标记状态不丢失

**标记用途**：
1. **疑难题目标记**：用户手动标记难理解的题目
2. **已掌握标记**：错题模式下标记已掌握的题目（与错题本"已掌握"功能联动）

**快捷键**：
- `空格键`：切换当前题目标记状态

## 🔧 技术细节

### 编号计算时机
1. 加载题目时：调用 `get_questions_with_serial()` 计算连续编号
2. 随机模式：打乱后重新编号（1-N）
3. 显示时：优先使用 `serial_number`，如果为 None 则使用 `index + 1`

### 标记更新流程
1. 用户点击"标记"按钮或按空格键
2. 更新内存中的 `marked_questions` 集合
3. 更新数据库 `practice_records` 表
4. 刷新答题卡显示

## ⚠️ 注意事项

1. **编号是动态计算的**：不是数据库字段，每次加载时重新计算
2. **删除题目后编号会变**：删除题目后，后续题目的序号会前移
3. **随机模式重新编号**：随机打乱后，序号从 1 重新排列
4. **标记与题型无关**：标记功能适用于所有题型

## 🎯 使用指南

### 查看连续编号
1. 选择左侧分类（如"单选题"）
2. 点击"▶ 开始练习"
3. 题目显示为 `[序号] 题目内容`

### 标记题目
1. 在答题界面点击"📌 标记"按钮
2. 或按空格键快速标记
3. 答题卡上标记题目显示 📌 标识

### 取消标记
1. 再次点击"📌 标记"按钮
2. 或按空格键取消标记
