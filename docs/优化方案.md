## 🔴 紧急修复（已完成）

### 1. 下一题按钮缺失 ✅ 已修复

**问题原因**：
- 布局中的 `addStretch()` 在按钮之前，导致按钮被挤压
- 按钮没有设置最小宽度，在某些窗口宽度下渲染异常
- 全局样式未正确处理禁用状态

**修复措施**：
- 为所有操作按钮设置固定宽度
- 在布局两侧添加弹性空间，保持居中对称
- 更新全局样式，添加 `min-width` 和更好的禁用状态样式

### 2. 选项文本重叠 ✅ 已修复

**问题原因**：
- `QRadioButton::indicator` 设置为 0x0 后，文本渲染位置仍尝试使用 indicator 空间
- 缺少足够的内边距和最小高度

**修复措施**：
- 完善 indicator 样式，确保完全隐藏
- 增加按钮最小高度到 50px
- 添加左内边距 `padding-left: 16px`
- 增加选项间距到 12px

## 🟡 阶段二：交互体验优化

### 1. 添加题目跳转功能

**当前状态**：只能通过上一题/下一题按钮逐题切换，效率低

**优化方案**：
- 添加题目编号输入框，支持直接跳转
- 添加题目列表侧边栏，显示已答/未答状态
- 支持 Ctrl+G 快捷键快速跳转

**实现建议**：
```python
# 在 question_widget.py 中添加
self.jump_input = QLineEdit()
self.jump_input.setPlaceholderText("题号")
self.jump_input.setFixedWidth(60)
self.jump_input.returnPressed.connect(self._jump_to_question)

def _jump_to_question(self):
    try:
        num = int(self.jump_input.text())
        if 1 <= num <= len(self.questions):
            self.current_index = num - 1
            self._show_question(self.current_index)
    except ValueError:
        pass
```

### 2. 添加答题进度指示

**当前状态**：只有进度条，无法区分已答/未答题目

**优化方案**：
- 在题目列表中显示答题状态（✅正确 / ❌错误 / ⏸未答）
- 添加颜色编码：绿色=正确，红色=错误，灰色=未答
- 支持按答题状态筛选题目

### 3. 答题后自动跳转下一题

**当前状态**：提交答案后需手动点击下一题

**优化方案**：
- 添加"自动下一题"选项开关
- 答对后延迟 1.5 秒自动跳转
- 答错后显示解析，等待用户手动操作

**实现建议**：
```python
# 在 _submit_answer 方法中添加
if is_correct and self.auto_next_enabled:
    QTimer.singleShot(1500, self._next_question)
```

### 4. 添加暂停/继续功能

**当前状态**：无法中断练习

**优化方案**：
- 添加"暂停"按钮，停止计时器
- 暂停时隐藏题目内容，防止偷看答案
- 显示暂停时长统计
