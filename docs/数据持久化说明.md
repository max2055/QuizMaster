# 数据持久化说明

## 📊 数据存储架构

### 数据库类型
QuizMaster 使用 **SQLite** 作为本地数据库，数据持久化存储在磁盘上。

### 数据库文件位置
```
QuizMaster/quizmaster.db
```
- 默认位于程序运行目录
- 单文件数据库，便于备份和迁移
- 无需额外配置，开箱即用

---

## 🗄️ 数据表结构

### 1. categories - 题库分类表
存储题目分类信息，支持多级分类（树形结构）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| name | TEXT | 分类名称 |
| parent_id | INTEGER | 父分类 ID（支持多级） |
| created_at | DATETIME | 创建时间 |

### 2. questions - 题目表
存储所有题目数据

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| category_id | INTEGER | 分类 ID（外键） |
| question_text | TEXT | 题目内容 |
| question_type | TEXT | 题型：single/multi/true_false/fill |
| options | TEXT | 选项（JSON 格式） |
| answer | TEXT | 正确答案 |
| explanation | TEXT | 答案解析 |
| difficulty | TEXT | 难度：easy/medium/hard |
| tags | TEXT | 标签（JSON 数组） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3. practice_records - 答题记录表
存储每次答题的详细记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| question_id | INTEGER | 题目 ID（外键） |
| user_answer | TEXT | 用户答案 |
| is_correct | BOOLEAN | 是否正确 |
| time_spent | INTEGER | 耗时（秒） |
| is_marked | BOOLEAN | 是否标记 |
| practice_session_id | INTEGER | 练习会话 ID（外键） |
| created_at | DATETIME | 创建时间 |

### 4. practice_sessions - 练习会话表
存储每次练习会话的汇总信息

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| mode | TEXT | 模式：sequence/random/wrong |
| category_id | INTEGER | 分类 ID |
| total_questions | INTEGER | 计划题量 |
| completed_questions | INTEGER | 已完成题数 |
| correct_count | INTEGER | 正确数 |
| start_time | DATETIME | 开始时间 |
| end_time | DATETIME | 结束时间 |
| status | TEXT | 状态：pending/running/completed |

### 5. wrong_questions - 错题本表
自动记录答错的题目

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| question_id | INTEGER | 题目 ID（外键） |
| wrong_count | INTEGER | 错误次数 |
| last_wrong_at | DATETIME | 最后错误时间 |
| mastered | BOOLEAN | 是否已掌握 |

## 💾 数据持久化机制

### 自动保存
- **题目数据**：添加/编辑/删除后立即保存到数据库
- **答题记录**：每次提交答案后立即保存
- **练习会话**：开始、进度更新、结束时自动保存
- **错题记录**：答错时自动加入错题本

### 事务处理
使用 SQLite 事务确保数据一致性：
```python
with self.db.get_cursor() as cursor:
    cursor.execute("INSERT INTO ...")
    # 自动 commit，异常时自动 rollback
```

### 连接管理
- 使用 `sqlite3.connect()` 创建数据库连接
- 设置 `check_same_thread=False` 支持多线程访问
- 程序关闭时自动关闭连接

## 🔄 进度记忆实现

### 题目进度
通过以下机制实现进度记忆：

1. **答题记录表** (`practice_records`)
   - 记录每道题的答题历史
   - 包含用户答案、正误、耗时等信息
   - 可通过 `question_id` 查询某题的所有答题记录

2. **练习会话表** (`practice_sessions`)
   - 记录每次练习的整体进度
   - 包含已完成题数、正确数等统计
   - 支持中断后恢复（后续版本）

3. **错题本表** (`wrong_questions`)
   - 自动记录答错的题目
   - 统计错误次数
   - 支持标记"已掌握"

### 查询示例

#### 查看某题的答题历史
```sql
SELECT * FROM practice_records 
WHERE question_id = 123 
ORDER BY created_at DESC;
```

#### 查看最近的练习会话
```sql
SELECT * FROM practice_sessions 
WHERE status = 'completed' 
ORDER BY end_time DESC 
LIMIT 10;
```

#### 查看错题本
```sql
SELECT q.*, w.wrong_count, w.last_wrong_at
FROM questions q
JOIN wrong_questions w ON q.id = w.question_id
WHERE w.mastered = 0
ORDER BY w.wrong_count DESC;
```

## 📊 统计数据来源

### 总体统计
- **总题量**: `SELECT COUNT(*) FROM questions`
- **已练习**: `SELECT COUNT(DISTINCT question_id) FROM practice_records`
- **正确率**: 根据 `practice_records.is_correct` 计算
- **错题本**: `SELECT COUNT(*) FROM wrong_questions WHERE mastered = 0`

### 练习会话统计
- 完成会话数
- 平均正确率
- 练习时长统计

## 🔒 数据安全

### 注意事项
1. **定期备份**: 建议定期备份数据库文件
2. **避免直接修改**: 不要直接用 SQLite 工具修改数据，可能导致不一致
3. **版本兼容**: 数据库结构变更时注意版本兼容

### 恢复数据
如果数据库损坏：
1. 从备份恢复 `quizmaster.db`
2. 或重新导入题目（CSV/Excel）

## 🛠️ 技术细节

### 数据库初始化
程序启动时自动：
1. 检查数据库文件是否存在
2. 创建必要的表结构
3. 建立数据库连接

### 数据模型
使用 Python dataclass 定义数据模型：
- `Category` - 分类
- `Question` - 题目
- `PracticeRecord` - 答题记录
- `PracticeSession` - 练习会话
- `WrongQuestion` - 错题

### 服务层封装
通过服务层封装数据库操作：
- `QuestionService` - 题目管理
- `PracticeService` - 练习管理
- `ImportService` - 导入服务
